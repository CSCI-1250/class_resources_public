📚 Intro to CS & Programming Study Guide

🔧 Algorithm Design

Definition: A systematic approach to creating step-by-step solutions for computational problems.
Key Components: Problem analysis, input/output identification, logical sequencing, efficiency considerations
Example: Designing an algorithm to find the largest number in a list:
1. Start with first number as current maximum 2. Compare each remaining number to current max 3. If larger, update maximum 4. Return final maximum

💻 Console Input/Output

Common Methods Across Languages:
Python: input(), print() C#: Console.ReadLine(), Console.WriteLine()
Do you know the differences between the input function and Console.Read()/Readline() ?

🧠 Types of Reasoning

Inductive Reasoning

Pattern: Specific → General

Example: Testing several prime numbers and noticing they're all odd (except 2), then hypothesizing "most primes are odd"

Deductive Reasoning

Pattern: General → Specific

Example: Knowing "all programming languages have syntax rules" and "Python is a programming language," therefore "Python has syntax rules"

🔀 Control Flow & Conditionals

Execution Order: Conditions are evaluated sequentially until one is true.
def categorize_temperature(temp): if temp > 100: return "Too hot!" elif temp > 80: return "Warm" elif temp > 60: return "Mild" else: return "Cool"
Trace Practice: Walk through each condition step-by-step with specific values

🔢 Set Operations

Union (|)
Combines all unique elements
{1,2} | {2,3} = {1,2,3}
Intersection (&)
Elements in both sets
{1,2} & {2,3} = {2}
Difference (-)
Elements in first, not second
{1,2} - {2,3} = {1}

⚡ Boolean Logic

Operation Symbol True when... Example
AND && or and Both operands true true && false = false
OR || or or At least one operand true true || false = true
NOT ! or not Operand is false !true = false
Order of Operations: Parentheses → NOT → AND → OR

🚗 Problem-Solving Framework

Computational Thinking Steps:

  1. Problem Understanding: Define inputs, outputs, and constraints
  2. Decomposition: Break complex problem into smaller sub-problems
  3. Pattern Recognition: Identify similarities to known problems
  4. Abstraction: Focus on essential features, ignore irrelevant details
  5. Algorithm Design: Create step-by-step solution
  6. Testing & Optimization: Verify correctness and improve efficiency

🔁 Loop Structures

While Loop Patterns

Python

counter = 1 while counter <= 5: print(f"Count: {counter}") counter += 1 # User input loop response = "" while response != "quit": response = input("Command: ")

C#

int counter = 1; while (counter <= 5) { Console.WriteLine($"Count: {counter}"); counter++; } // User input loop string response = ""; while (response != "quit") { response = Console.ReadLine(); }

⚙️ Function Fundamentals

Function Components

Method Signature: The first line defining function name and parameters
Parameters: Variable names in the function definition
Arguments: Actual values passed when calling the function
Return Statement: Sends a value back to the caller
Function Body: The code that executes when function is called
def calculate_average(score1, score2): # Method signature total = score1 + score2 # Function body average = total / 2 # Function body return average # Return statement result = calculate_average(85, 92) # Arguments: 85, 92

⚠️ Common Syntax Differences

Python Syntax

• Uses colons (:) after control statements
• Indentation defines code blocks
• No semicolons needed
• Dynamic typing
• Case sensitive variable names

C# Syntax

• Uses curly braces { } for code blocks
• Semicolons end statements
• Parentheses required for conditions
• Static typing required
• Case sensitive everything

📝 Study Tips

  • Practice tracing: Work through code line-by-line with different inputs
  • Draw it out: Use diagrams for set operations and boolean logic
  • Break down problems: Start with simple cases, then add complexity
  • Test edge cases: Consider boundary values and special conditions
  • Explain your thinking: Verbalize your problem-solving process
  • Know the syntax: Practice common syntax differences between languages
  • Understand types: Remember when explicit type conversion is needed
