‚úÖ Evaluating Complex Boolean Conditions

Order of Operations: Parentheses ‚Üí ! (NOT) ‚Üí && (AND) ‚Üí || (OR)
Example 1: Evaluate (true || false) && true || true
Step 1: (true || false) = true Step 2: true && true = true Step 3: true || true = true Final Result: true
Example 2: Evaluate !false && (true || false)
Step 1: !false = true Step 2: (true || false) = true Step 3: true && true = true Final Result: true
Example 3: Practical scenario
int age = 25; bool hasLicense = true; bool isWeekend = false; // Can rent a car if: age >= 21 AND has license AND it's weekend bool canRent = (age >= 21) && hasLicense && isWeekend; // Result: true && true && false = false

üîÅ Loop Structures

While Loop (Pre-test Loop)

// Condition checked BEFORE entering loop // May execute 0 times if condition is false initially int count = 0; while (count < 5) // Pre-test { Console.WriteLine(count); count++; // Incrementing control variable }
Control Variable: A variable that determines how many times a loop executes. It's initialized before the loop, tested in the condition, and modified within the loop body.

Do-While Loop (Post-test Loop)

// Condition checked AFTER executing loop body // ALWAYS executes at least once int count = 0; do { Console.WriteLine(count); count--; // Decrementing control variable } while (count > 0); // Post-test
Key Difference: do-while is guaranteed to run at least once, even if the condition is initially false!

For Loop

// Compact syntax with initialization, condition, and update for (int i = 0; i < 10; i++) // i is the control variable { Console.WriteLine(i); } // Counting down for (int i = 10; i >= 0; i--) { Console.WriteLine(i); }

Foreach Loop

// Iterates through each element in a collection int[] scores = {85, 90, 78, 92, 88}; foreach (int score in scores) { Console.WriteLine(score); } List<string> names = new List<string> {"Alice", "Bob", "Charlie"}; foreach (string name in names) { Console.WriteLine(name); }
Purpose of Foreach: Provides a clean, readable way to iterate through all elements in a collection without managing index variables. You cannot modify the collection or access the index directly within a foreach loop.

üèóÔ∏è Object-Oriented Programming Concepts

Classes vs Objects

Class: A blueprint or template that defines the structure and behavior
Object: A specific instance created from that class
// Class definition (the blueprint) public class Car { public string Make { get; set; } public string Model { get; set; } public int Year { get; set; } public void Start() { Console.WriteLine($"{Make} {Model} is starting..."); } } // Creating objects (instances) from the class Car myCar = new Car(); // Instance 1 myCar.Make = "Toyota"; myCar.Model = "Camry"; Car yourCar = new Car(); // Instance 2 (separate from myCar) yourCar.Make = "Honda"; yourCar.Model = "Accord";
Class Members: Belong to the class itself (static)
Instance Members: Belong to each individual object

Static vs Instance: Static members are shared across all instances and accessed via the class name (Math.Sqrt()). Instance members are unique to each object and accessed via object references (myCar.Start()).

public class Student { // Fields private string name; private int id; private double gpa; // Default constructor public Student() { name = "Unknown"; id = 0; gpa = 0.0; } // Parameterized constructor public Student(string studentName, int studentId, double studentGpa) { name = studentName; id = studentId; gpa = studentGpa; } // Using constructors Student student1 = new Student(); // Uses default Student student2 = new Student("Ryan", 12345, 3.8); // Uses parameterized }
Constructor Purpose: Special method called automatically when creating an object. Initializes the object's state (sets initial values for fields).

üíæ Memory Management: Stack vs Heap

Stack Memory

Stores: Value types (int, double, bool, char, struct)
Characteristics:
‚Ä¢ Fast access
‚Ä¢ Automatic memory management
‚Ä¢ Limited size
‚Ä¢ Variables exist only in their scope
int x = 5; // Stored on stack double y = 3.14; // Stored on stack bool flag = true; // Stored on stack

Heap Memory

Stores: Reference types (objects, arrays, strings)
Characteristics:
‚Ä¢ Larger capacity
‚Ä¢ Managed by garbage collector
‚Ä¢ Slower access than stack
‚Ä¢ Objects persist until no references remain
Student s = new Student(); // Object on heap, reference on stack string name = "Ryan"; // String on heap, reference on stack int[] scores = new int[5]; // Array on heap, reference on stack

Reference Type Behavior

// Reference types point to memory locations Student student1 = new Student("Alice", 101, 3.5); Student student2 = student1; // Both refer to SAME object student2.Name = "Bob"; Console.WriteLine(student1.Name); // Prints "Bob" - same object! // Compare to value types int a = 5; int b = a; // b gets a COPY of the value b = 10; Console.WriteLine(a); // Prints 5 - separate variables!
Important: When you assign one reference variable to another, you're copying the reference (address), not the object. Both variables point to the same object in memory.

üìù Overriding ToString()

public class Student { public string Name { get; set; } public int ID { get; set; } public double GPA { get; set; } // Override ToString() for custom string representation public override string ToString() { return $"Student: {Name}, ID: {ID}, GPA: {GPA:F2}"; } } // Using ToString() Student s = new Student { Name = "Ryan", ID = 12345, GPA = 3.85 }; Console.WriteLine(s); // Automatically calls ToString() // Output: Student: Ryan, ID: 12345, GPA: 3.85 string info = s.ToString(); // Explicitly call ToString()
Purpose: Provides a readable string representation of an object. Useful for debugging and displaying object information. Every object inherits ToString() from the Object class, but you can override it for custom behavior.

// Read, process, and write back string[] lines = File.ReadAllLines("scores.txt"); List<string> processed = new List<string>(); foreach (string line in lines) { // Process each line int score = int.Parse(line); processed.Add($"Score: {score}, Grade: {CalculateGrade(score)}"); } File.WriteAllLines("results.txt", processed);
Remember: File I/O can throw exceptions! In production code, you should use try-catch blocks to handle potential errors (file not found, permission denied, etc.).

üìö Arrays vs Lists

Arrays

// Declaration and initialization int[] scores = new int[5]; // Fixed size // With initial values int[] numbers = {1, 2, 3, 4, 5}; string[] names = new string[3] {"Alice", "Bob", "Charlie"}; // Accessing elements scores[0] = 95; scores[1] = 87; // Size property int size = scores.Length;
Arrays:
‚Ä¢ Fixed size
‚Ä¢ Use Length property
‚Ä¢ Better performance
‚Ä¢ Cannot add/remove elements

Lists

// Must import: using System.Collections.Generic; // Declaration and initialization List<int> scores = new List<int>(); // With initial values List<string> names = new List<string> {"Alice", "Bob"}; // Adding elements scores.Add(95); scores.Add(87); // Size property int size = scores.Count;
Lists:
‚Ä¢ Dynamic size
‚Ä¢ Use Count property
‚Ä¢ More flexibility
‚Ä¢ Can Add(), Remove(), Insert()
Common Mistake: Using .Length on a List or .Count on an array!
Arrays use Length (property, no parentheses)
Lists use Count (property, no parentheses)

Common List Methods

List<int> numbers = new List<int> {10, 20, 30, 40}; numbers.Add(50); // Adds to end: {10, 20, 30, 40, 50} numbers.Remove(20); // Removes first 20: {10, 30, 40, 50} numbers.RemoveAt(0); // Removes at index 0: {30, 40, 50} numbers.Insert(1, 35); // Insert at index 1: {30, 35, 40, 50} bool has40 = numbers.Contains(40); // true numbers.Clear(); // Removes all elements

üéØ Why Use Methods and Classes?

Benefits of Methods

1. Focused Responsibilities
Each method does one specific task, making code easier to understand
// Instead of one giant method... public static void ProcessStudent() { // Calculate grade // Validate input // Format output // Save to file } // Break into focused methods public static double CalculateGrade(int score) { } public static bool ValidateScore(int score) { } public static string FormatOutput(double grade) { } public static void SaveToFile(string data) { }
2. Code Reuse
Write logic once, use it many times
public static bool IsEven(int number) { return number % 2 == 0; } // Use it everywhere without rewriting if (IsEven(score)) { } if (IsEven(age)) { } if (IsEven(count)) { }
3. Easier Modification
Change behavior in one place, affects all uses
// Change how grade is calculated in ONE method public static char CalculateLetterGrade(double percent) { // Update this logic once, affects entire program if (percent >= 90) return 'A'; // ... }

Benefits of Classes

1. Encapsulation
Bundle related data and methods together, hide implementation details
public class BankAccount { private double balance; // Hidden from outside public void Deposit(double amount) // Controlled access { if (amount > 0) balance += amount; } }
2. Organization
Group related functionality logically
// All student-related logic in one place public class Student { public string Name { get; set; } public double CalculateGPA() { } public bool IsHonorsStudent() { } public void PrintTranscript() { } }
3. Reusability and Scalability
Create multiple instances with different data
Student alice = new Student("Alice", 3.8); Student bob = new Student("Bob", 3.5); Student charlie = new Student("Charlie", 3.9); // Each object maintains its own data
