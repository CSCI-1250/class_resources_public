Understanding Integer vs Decimal Division

üî¢ Integer Division:

int result1 = 10 / 3;        // Result: 3 (truncated)
int result2 = 9 / 4;         // Result: 2 (truncated)
int result3 = 7 / 2;         // Result: 3 (not 3.5!)

Important: Integer division always truncates (cuts off) the decimal part!

üí´ Decimal Division:

double result3 = 10.0 / 3.0; // Result: 3.3333...
double result4 = 10 / 3.0;   // Result: 3.3333... (mixed types)
double result5 = 10.0 / 3;   // Result: 3.3333... (mixed types)

Key: At least one operand must be decimal type!

When Math Gets Weird

ü§Ø The Problem:

double result = 1.4 + 1.2;
Console.WriteLine(result);  // Might print: 2.5999999999999996

üîç Why This Happens:

  • Computers use binary (base-2)
  • Some decimal numbers can't be represented exactly
  • Similar to how 1/3 = 0.333... in decimal
  • It's a fundamental limitation, not a bug

üí° Solutions:

decimal precise = 1.4m + 1.2m;        // Use decimal for exact math
double rounded = Math.Round(result, 2); // Round for display
bool isClose = Math.Abs(result - 2.6) < 0.001; // Compare with tolerance

‚ö†Ô∏è Important: Always use decimal for financial calculations where exact precision matters!
