Object Classes in C#
From Static to Object-Oriented Programming

Today's Goals:

  1. Understand the difference between static and object classes
  2. Learn how to design and create your own object classes
  3. Practice building constructors and properties
  4. Explore encapsulation and data hiding
  5. Connect objects in complex applications
Static Class vs Object Class

Static Class

  • Cannot be instantiated (no objects)
  • Call methods directly on the class
  • All class members shared globally
  • Use for utility functions and constants

static class Calculator
{
    static int Add(int a, int b)
    {
        return a + b;
    }
}

// Using it:
int result = Calculator.Add(5, 3);

Object Class

  • Must be instantiated to use (create objects)
  • Call methods on specific instances
  • Each object has its own data
  • Use for representing real-world entities

class Person
{
    public string Name;
    
    public void SayHello()
    {
        Console.WriteLine($"Hello, I'm {Name}");
    }
}

// Using it:
Person student = new Person();
student.Name = "Alex";
student.SayHello();
What it Means to Instantiate

Creating Objects from a Class Blueprint

Instantiation is the process of creating an object (an instance) from a class. The class serves as a blueprint or template, and the object is the actual "thing" created from that blueprint.


// Class definition (blueprint)
class Dog
{
    public string Name;
    public string Breed;
    
    public void Bark()
    {
        Console.WriteLine($"{Name} says: Woof!");
    }
}

// Creating instances (actual dogs)
Dog fido = new Dog();      // Instantiation
fido.Name = "Fido";        // Setting properties
fido.Breed = "Labrador";
fido.Bark();               // Using methods

Dog rex = new Dog();       // Another instance
rex.Name = "Rex";          // Different data
rex.Breed = "German Shepherd";
rex.Bark();

üìù Key Points About Instantiation

  • The new keyword creates an instance in memory
  • Each instance has its own copy of fields/properties
  • Multiple objects can be created from the same class
  • Objects can have different data but share the same behavior
Instantiation Example: Using the Random Class

// Creating a Random object
Random rand = new Random();   // Instantiation

// Using the object's methods
int number1 = rand.Next(1, 101);  // Random number between 1-100
int number2 = rand.Next(1, 101);  // Different random number
double value = rand.NextDouble();  // Random decimal between 0-1

// Creating another Random instance
Random anotherRand = new Random(42);  // Seed of 42
int number3 = anotherRand.Next(1, 101);

üîç What's Happening?

  • Random is a built-in C# class
  • We create instances using new Random()
  • Each Random object maintains its own internal state
  • We call methods like Next() on our instances

Why Use Objects?

  • Each Random instance generates its own sequence
  • They can be configured differently (different seeds)
  • Can be passed to methods, stored in variables
  • Encapsulates complex functionality in a simple interface

Accessors and Modifiers: Introduction to Encapsulation

What is Encapsulation?

Encapsulation is the bundling of data (attributes) and methods (behaviors) that work on the data, while restricting direct access to some of the object's components. It's one of the four fundamental OOP concepts.

üõ°Ô∏è Why Encapsulate Data?

  • Protect data from unauthorized access
  • Prevent invalid data assignments
  • Hide implementation details
  • Control how data is accessed and modified
  • Make code more maintainable

üîë Access Modifiers

  • public: Accessible from anywhere
  • private: Only accessible within the class
  • protected: Accessible in class and derived classes
  • internal: Accessible within the same assembly

Best Practice

Make fields private and provide public methods or properties to access them when needed. This ensures data integrity and encapsulation.

Getters and Setters: Traditional Methods

Problem with Public Fields:


public class Person
{
    public int Age;  // Public field - no control
}

Person p = new Person();
p.Age = -30;  // Invalid age! But allowed

Solution with Getter/Setter Methods:


public class Person
{
    private int age;  // Private field
    
    // Getter method
    public int GetAge()
    {
        return age;
    }
    
    // Setter method with validation
    public void SetAge(int value)
    {
        if (value >= 0 && value < 120)
            age = value;
        else
            throw new ArgumentException("Invalid age");
    }
}

Person p = new Person();
p.SetAge(25);      // Valid age
int age = p.GetAge();
// p.SetAge(-30);  // Throws exception
Getters and Setters: Traditional Methods

Key Benefits of Getters/Setters

  • Encapsulation: Hide implementation details
  • Validation: Check data before accepting it
  • Control: Restrict how data can be modified
  • Flexibility: Change implementation without affecting clients

Common Patterns

  • Read-only: Provide getter but no setter
  • Write-only: Provide setter but no getter (password fields)
  • Lazy calculation: Compute value only when getter is called
  • Derived value: Calculate from other fields
Properties in C#: Simplified Accessors

Properties: C#'s Elegant Accessor Syntax

C# provides a simplified syntax for creating getters and setters called "properties". They achieve the same encapsulation but with less code.

Traditional Getter/Setter Methods:


public class Person
{
    private int age;
    
    public int GetAge()
    {
        return age;
    }
    
    public void SetAge(int value)
    {
        if (value >= 0 && value < 120)
            age = value;
        else
            throw new ArgumentException("Invalid age");
    }
}

// Usage
Person p = new Person();
p.SetAge(25);
int age = p.GetAge();

C# Properties:


public class Person
{
    private int age;  // Backing field
    
    // Property with same functionality
    public int Age
    {
        get { return age; }  // Getter
        set                  // Setter
        {
            if (value >= 0 && value < 120)
                age = value;
            else
                throw new ArgumentException("Invalid age");
        }
    }
}

// Usage
Person p = new Person();
p.Age = 25;        // Calls set
int age = p.Age;   // Calls get
Properties in C#: Simplified Accessors

Property Benefits

  • Cleaner, more concise syntax
  • Same encapsulation as traditional methods
  • More intuitive property-like usage
  • Compatible with interfaces and inheritance
  • Can be used with attributes and reflection

The Class Constructor

What is a Constructor?

A constructor is a special method that is automatically called when an object is created. Its primary purpose is to initialize the object's fields and ensure the object starts in a valid state.


public class ChatSession
{
    public string ID { get; set; }
    public string ClientName { get; set; }
    public string InitialMessage { get; set; }
    public DateTime CreatedAt { get; private set; }
    public string Status { get; set; }
    
    // Constructor
    public ChatSession()
    {
        // Initialize with default values
        ID = "New";
        CreatedAt = DateTime.Now;
        Status = "Pending";
    }
}

// Using the constructor
ChatSession chat = new ChatSession();
// Fields already initialized: ID is "New", CreatedAt is now, Status is "Pending"
Console.WriteLine(chat.CreatedAt);  // Current time when created

üìù Key Points About Constructors

  • Same name as the class
  • No return type (not even void)
  • Automatically called when using new
  • Initializes object to a usable state
  • Can have multiple constructors with different parameters

Design Decisions

  • Private fields: Protect sensitive data from direct access
  • Properties with validation: Control how data is set and ensure valid values
  • Read-only MII: Should be determined from the card number, not set directly
  • Validation methods: Check if the card is expired or valid
CreditCard Implementation

public class CreditCard
{
    // Private fields
    private string cardNumber;
    private string cardholderName;
    private int expirationMonth;
    private int expirationYear;
    private string securityCode;
    private int mii; // Major Industry Identifier
    
    // Properties with validation
    public string CardNumber 
    { 
        get { return cardNumber; }
        private set 
        {
            if (IsValidCardNumber(value))
            {
                cardNumber = value;
                SetMIIFromCardNumber(value);
            }
            else
                throw new ArgumentException("Invalid card number");
        }
    }
    
    public string CardholderName 
    { 
        get { return cardholderName; }
        set 
        {
            if (!string.IsNullOrEmpty(value))
                cardholderName = value;
            else
                throw new ArgumentException("Cardholder name cannot be empty");
        }
    }
    
    public int ExpirationMonth 
    { 
        get { return expirationMonth; }
        set 
        {
            if (value >= 1 && value <= 12)
                expirationMonth = value;
            else
                throw new ArgumentException("Month must be between 1 and 12");
        }
    }
    
    public int ExpirationYear 
    { 
        get { return expirationYear; }
        set 
        {
            if (value >= DateTime.Now.Year)
                expirationYear = value;
            else
                throw new ArgumentException("Year cannot be in the past");
        }
    }
    
    public string SecurityCode 
    { 
        get { return securityCode; }
        set 
        {
            if (value.Length >= 3 && value.Length <= 4)
                securityCode = value;
            else
                throw new ArgumentException("Security code must be 3 or 4 digits");
        }
    }
    
    // Read-only property for MII
    public int MII 
    { 
        get { return mii; }
    }
CreditCard Implementation (Continued)

    // Constructor
    public CreditCard(string cardNumber, string cardholderName, 
                     int expMonth, int expYear, string cvv)
    {
        // Properties handle validation
        CardNumber = cardNumber; // This also sets MII
        CardholderName = cardholderName;
        ExpirationMonth = expMonth;
        ExpirationYear = expYear;
        SecurityCode = cvv;
    }
    
    // Check if card is expired
    public bool IsExpired()
    {
        DateTime now = DateTime.Now;
        return (ExpirationYear < now.Year) || 
               (ExpirationYear == now.Year && ExpirationMonth < now.Month);
    }
    
    // Get card type based on MII
    public string GetCardType()
    {
        switch(MII)
        {
            case 3: return "American Express";
            case 4: return "Visa";
            case 5: return "MasterCard";
            case 6: return "Discover";
            default: return "Unknown";
        }
    }
    
    // Sets the MII from card number (first digit)
    private void SetMIIFromCardNumber(string cardNumber)
    {
        if (!string.IsNullOrEmpty(cardNumber) && cardNumber.Length > 0)
        {
            mii = int.Parse(cardNumber[0].ToString());
        }
    }
    
    // Helper method to validate card number (simplified)
    private bool IsValidCardNumber(string number)
    {
        return !string.IsNullOrEmpty(number) && 
               number.All(char.IsDigit) &&
               number.Length >= 13 && 
               number.Length <= 19;
    }
    
    // Display card information
    public void DisplayInfo()
    {
        string lastFour = CardNumber.Substring(CardNumber.Length - 4);
        Console.WriteLine($"Card: {GetCardType()} ending in {lastFour}");
        Console.WriteLine($"Cardholder: {CardholderName}");
        Console.WriteLine($"Expires: {ExpirationMonth}/{ExpirationYear}");
        Console.WriteLine($"Status: {(IsExpired() ? "Expired" : "Valid")}");
    }
}

How Objects Work Together

üîÑ Objects in the Real World

In real applications, objects don't exist in isolation. They interact with each other to form complex systems, just like objects in the real world.

Customer
- name: string
- email: string
- phone: string
- creditCards: List<CreditCard>
+ AddCreditCard(CreditCard card): void
+ GetDefaultCard(): CreditCard
+ ChargeAmount(decimal amount): bool

public class Customer
{
    public string Name { get; set; }
    public string Email { get; set; }
    public string Phone { get; set; }
    private List creditCards;
    
    public Customer(string name, string email, string phone)
    {
        Name = name;
        Email = email;
        Phone = phone;
        creditCards = new List();
    }
    
    public void AddCreditCard(CreditCard card)
    {
        creditCards.Add(card);
    }
    
    public CreditCard GetDefaultCard()
    {
        if (creditCards.Count > 0)
            return creditCards[0];
        return null;
    }
    
    public bool ChargeAmount(decimal amount)
    {
        CreditCard card = GetDefaultCard();
        if (card != null && !card.IsExpired())
        {
            Console.WriteLine($"Charging ${amount} to {card.GetCardType()} ending in " + 
                             card.CardNumber.Substring(card.CardNumber.Length - 4));
            return true;
        }
        return false;
    }
}
