What is a Data Structure?

Data Structure

A method of storing data in a computer

  • Ideally, with easy, fast access
  • Like a filing cabinet or shelves in a library – organized with labels
  • Some examples include arrays, lists, hash maps, dispatch tables
Why Use Data Structures?
  • Scalability – Programs quickly become quite large in scale, as does the data within them. Holding that data is difficult – accessing it moreso
  • Organization – Data structures, if used correctly, are an easy way to organize that data into a readable, maintainable, singular structure
  • Efficiency – The correct data structure can make accessing a piece of data nearly instantaneous (more about complexity theory later)
Arrays
  • Arrays are the simplest, most common data structure
  • Collection of items, of the same type, stored in a specific order
  • Items labeled in relation to their position within the array
  • Allocates a section of memory in the computer to store this information
  • Great at storing sets of numbers

Basics of Sets

Set Definition

  • Set – an unordered collection of unique items
  • Items in a set are called elements
  • Ex: Set of prime numbers < 10...
  • P = { 2, 3, 5, 7 }

Set Properties

  • Unique: { 1, 2, 3 } = { 1, 1, 2, 2, 3 }
  • Unordered: { c, a, b } = { a, b, c }
  • Almost anything can be defined as a set
Set Operations

Union:

A = { 1, 3, 5, 7 }

B = { 2, 4, 6, 8 }

A ∪ B = { 1, 2, 3, 4, 5, 6, 7, 8 }

Intersection:

A = { 1, 2, 3, 4 }

B = { 2, 4 }

A ∩ B = { 2, 4 }

Set Operations (cont.)

Difference:

A = { 1, 3, 5, 7 }

B = { 1, 2, 3, 4 }

A - B = { 5, 7 }

B - A = { 2, 4 }

Complement:

U = { 10, 20, 30, 40, 50 }

P = { 20, 30 }

PC = U – P = { 10, 40, 50 }

Basics of Number Systems

Number Systems

Representing numbers using symbols in a consistent manner

  • Has a base (or radix) – Total number of unique digits used to represent numbers
  • Decimals – 1, 2, 3, 4
  • Roman Numerals – I, II, III, IV
  • Binary – 0001, 0010, 0011, 0100
Decimal Number System
  • Decimal (base 10) – 1, 2, 3, 4, 5, etc.
  • Position of each digit determines its value (Place value)
  • Each place value is a power of 10, so the number 132 is:

132 = 1×10² + 3×10¹ + 2×10⁰

= 100 + 30 + 2 = 132

Easily readable by humans, not so much by computers...

Binary Number System
  • Binary (base 2) – 0001, 0010, 0011, 0100
  • Only two digits – 0, and 1
  • Represents voltage state: 0 = off, no voltage; 1 = on, voltage present
  • Each binary digit is called a bit, a series of bits is data
  • A series of 1011 is a series of four bits
  • Every program that runs on your computer is a long series of bits
Converting Decimal to Binary

1011, from our previous example, can be represented as so:

1×2³ + 0×2² + 1×2¹ + 1×2⁰

8 + 0 + 2 + 1 = 11

As a reminder, you're working in powers of 2, since binary is base 2

Functions

Function

A user-defined, reusable block of code that accomplishes a certain task

  • print() – writes to the console
  • float() – converts an input to a floating point number
  • Given some input, an output is produced (think back to Von Neumann architecture)
Why Use Functions?
  • Modularity: Break down complex problems into smaller, more manageable pieces.
  • Reusability: You can use the same function multiple times without rewriting the code.
  • Readability: Make your code cleaner and easier for you and others to understand.
  • Allow us to convert/map data from one set to another
  • Domain – input of the function
  • Codomain – output of the function
Anatomy of a Function

Functions have three main components:

  1. Function Name: This specifies the unique name of the function
  2. Parameters: Variables that are placeholders for the information the function needs to work. These are inside the parentheses ()
  3. Body: The code block inside the function that contains the instructions
Anatomy of a Function

Let's say we want to create a function in Python to calculate the area of a rectangle:

def calculate_area(length, width): area = length * width return area

Here, the parameters are the length and width of the rectangle, the body multiplies them together, and it returns the area

Calling a Function

Calling a function is the process of using the function, in code, for its intended purpose

We call, or invoke, functions using their parameters as described during their definition:

# For a rectangle that's 5x10... result = calculate_area(5, 10) # To print that to the console... print(calculate_area(5, 10))
Return Values

Functions, as a structure, can perform an action or return a value

A return statement gives a value back to the code that called the function

This is useful when you need to use the result of a function's task:

def add_numbers(a, b): return a + b result = add_numbers(3, 4)
Return Values (cont.)

If we don't want to return a value, we can omit the return statement or return None.

Useful when we want to simply perform an action, and finish:

def greet(name): print(f"Hello, {name}!") greet("Alice")
Recursion
  • Sometimes, you want a function to call itself.
  • Recursion - The process of a function calling itself
  • A lot like Russian nesting dolls - each doll contains a smaller version of itself.
  • Every recursive function needs a base case, a condition that stops the recursion and prevents an infinite loop.
  • It's often used for problems that can be broken down into smaller, self-similar sub-problems.
Loops

What are Loops?

  • Say you want to do something over and over again...
  • For example, watering 10 plants
  • You'd water the first, then the second, then the third, sequentially, until all 10 plants were watered
  • If you quit on the first, you'd have 9 unwatered plants

Loop Definition

Loop – a control structure that executes a block of code repeatedly until a certain condition is met

Why use Loops?

Benefits

  • Efficiency: Save you from writing the same code over and over again.
  • Automation: Essential for handling large collections of data, such as items in a list or characters in a string.
  • Flexibility: You can perform an action a specific number of times or until a condition changes.

Bad Example

You don't want something like this!

if number == 1: return False elif number == 2: return True elif number == 3: return False # ... and so on
Loop Types – The 'for' Loop

The 'for' loop is ideal when you know exactly how many times you want to repeat an action before your loop begins.

Are formatted with a variable, followed by an iterable:

for i in range(5): print(i)
Loop Types – The 'while' Loop

When you don't know how many iterations you need to do before the loop begins

Continues as long as the statement in the loop condition is 'True'

If you wanted to double a number until it's greater than 100...

number = 5 while number <= 100: number = number * 2 print(number)

If the condition starts as false, however, the loop will never run

Loop Types – The 'for' Loop with Lists

When iterating over elements (like those in a list), you can use a for loop

Automatically handles the starting and termination of the loop:

fruits = ["Apple", "Banana", "Cherry"] for fruit in fruits: print(fruit)
Conditional Statements – Branching

What are Conditionals?

  • Conditional statements allow your program to make decisions.
  • Execute different blocks of code based on whether a condition is true or false.

Types

  • 'if' Statement: The code inside the if block only runs if the condition is true.
  • 'else' Statement: The code inside the else block runs if the if condition is false.
  • 'elif' Statement: Checks for an additional condition if the first if statement is false.
Conditionals – if/else

Example: Checking whether a student passed or failed:

score = 75 if score >= 60: print("You passed!") else: print("You failed.")
Conditionals – if/else (cont.)

This can be extended almost indefinitely – there isn't really a limit as to how many if/else statements you can make.

Especially with the 'elif' block, which produces what is essentially another 'if' statement should the first not be true:

Conditionals – 'elif'
score = 88 if score >= 90: print("Grade: A") elif score >= 80: print("Grade: B") elif score >= 70: print("Grade: C") else: print("Grade: F")
Conditionals – Dictionary Alternative

If you have a lot of 'elif' statements, it may be cleaner to use a dictionary for lookup.

This is especially true if you're testing a single variable against a range of values

day = 3 days = { 1: "Monday", 2: "Tuesday", 3: "Wednesday" } print(days.get(day, "Another day"))
Nested Loops

What are Nested Loops?

  • What if you want a loop inside a loop?
  • This is accomplished using nested loops.
  • Used when you need to iterate over multiple sequences
  • When a task requires repeated actions within another repeating process, nested loops are also useful.

Use Cases

  • Useful when creating multidimensional data structures like matrices.
  • Processing 2D grids or tables
  • Generating patterns
Nested Loops
for i in range(4): for j in range(4): print("*", end="") print() # New line after each row
Nested Conditionals

Just like loops, you can also place conditional statements inside other conditional statements.

  • This allows for very specific decision-making based on multiple criteria at the same time.
  • A nested if statement is only evaluated if its parent if statement is true.
Nested Conditionals
age = 18 has_permission = True if age >= 18: print("You are old enough to enter.") if has_permission: print("Permission granted. Welcome!") else: print("You need permission to enter.") else: print("You are not old enough to enter.")
Conditionals – Prediction
  • If we can predict the outcome of these branched conditions, we can make execution of a program much, much faster.
  • Modern processors can look hundreds of instructions into the future, just by looking back at what previous branches it took.
  • Though not inherently guaranteed, a 4KiB program running on an Apple M1 can successfully predict 95+% of all the branches the code will take
  • This allows computers to potentially process entire chunks of a program ahead of time and is a lot of the reason that modern CPUs have gotten so fast.
