Void vs Value-Returning Methods

ğŸš« Void Methods

  • Don't return a value
  • Perform an action (display, modify, etc.)
  • Called for their side effects
  • Use Console.WriteLine, file operations

// VOID METHOD - Does something, returns nothing
static void PrettyPrint(string text)
{
    Console.WriteLine("*********************************");
    Console.WriteLine("* " + text);
    Console.WriteLine("*********************************");
    // No return statement needed
}

ğŸ“¤ Value-Returning Methods

  • Return a specific data type
  • Calculate and provide results
  • Can be used in expressions
  • Must have return statement
static int GetPlayerDamage(string characterClass)
{
    Random rand = new Random();
    
    if (characterClass == "Warrior")
        return rand.Next(8, 16);  // 8-15 damage
    else if (characterClass == "Mage")
        return rand.Next(6, 13);  // 6-12 damage
    else if (characterClass == "Rogue")
        return rand.Next(5, 11);  // 5-10 damage
    else
        return 1; // Default minimal damage
}

Method Call Chain - How Methods Work Together

ğŸ”— Understanding the Flow

Let's trace through what happens when the program runs, step by step:


// Let's trace this line by line:
static void Main(string[] args)
{
    // 1. Main calls PrettyPrint with a string
    PrettyPrint("Welcome to the Mystical Fortune Teller!");
    
    // 2. Main calls GetUserStringInput, stores result in userName
    string userName = GetUserStringInput("What is your name? ");
    
    // 3. Main calls PrettyPrint again, using userName variable
    PrettyPrint($"Hello, {userName}!");
    
    // 4. Main calls DisplayMenu (which calls PrettyPrint internally!)
    DisplayMenu();
    
    // 5. Main calls GetUserIntInput, stores result in choice
    int choice = GetUserIntInput("Select your fortune: ");
    
    // 6. Main calls GetFortune with choice, stores result in fortune
    string fortune = GetFortune(choice);
    
    // 7. Main calls PrettyPrint with the fortune
    PrettyPrint(fortune);
}

// Remember: Each method must exist for this to work!
// The order of method definitions doesn't matter,
// but they all must be present in your program.
// Call sequence with user input "Alice" and choice "2":

1. PrettyPrint() displays welcome box
2. GetUserStringInput() returns "Alice"
3. PrettyPrint() displays "Hello, Alice!"
4. DisplayMenu() shows options

5. GetUserIntInput() returns 2
6. GetFortune(2) returns career fortune

ğŸ§  Key Insight

Methods can call other methods! DisplayMenu() calls PrettyPrint() internally. This is called method composition - building complex behaviors from simpler ones.

Common Mistakes and How to Avoid Them

âŒ Mistake 1: Wrong Return Type


// WRONG: Method signature says string, but returns nothing
static string GetUserStringInput(string prompt)
{
    Console.Write(prompt);
    Console.ReadLine(); // Missing return!
}

// CORRECT: Actually return the value
static string GetUserStringInput(string prompt)
{
    Console.Write(prompt);
    return Console.ReadLine();
}

âŒ Mistake 2: Not Using Parameters


// WRONG: Ignoring the parameter
static void PrettyPrint(string text)
{
    Console.WriteLine("Hello World!"); // Hardcoded!
}

// CORRECT: Using the parameter
static void PrettyPrint(string text)
{
    Console.WriteLine("*** " + text + " ***");
}

    Console.WriteLine(text);
